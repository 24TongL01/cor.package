Introduction

The cor.package provides efficient and flexible tools for:

Calculating correlation coefficients using customizable methods (cor_new). Handling missing values in datasets (handle_na). Computing and visualizing correlation matrices for high-dimensional data (efficient_corr_matrix). Calculating correlation coefficients more efficiently using Rcpp (cor_new_rcpp_wrapper). This guide demonstrates how to use these functions through detailed examples and explanations.

cor_new(): Flexible Correlation Calculation

The cor_new function allows for custom correlation calculations with options for different methods and handling of missing data.

# Example data
x = c(1, 2, 3, NA, 5)
y = c(2, 4, NA, 8, 10)

# Pearson correlation with mean imputation
cor_new(x, y, method = "pearson", na_handle = "mean")
#> [1] 0.9354143

# Spearman correlation with complete case handling
cor_new(x, y, method = "spearman", na_handle = "complete")
#> [1] 1

# Kendall correlation
cor_new(x, y, method = "kendall")
#> [1] 0.1

handle_na(): Missing Value Handling

The handle_na function processes missing values in datasets based on user-defined methods.

# Example data
x = c(1, 2, 3, NA, 5)
y = c(2, 4, NA, 8, 10)

# Remove rows with missing values
handle_na(x, y, method = "complete")
#> $x
#> [1] 1 2 5
#> 
#> $y
#> [1]  2  4 10

# Impute missing values with the mean
handle_na(x, y, method = "mean")
#> $x
#> [1] 1.00 2.00 3.00 2.75 5.00
#> 
#> $y
#> [1]  2  4  6  8 10

efficient_corr_matrix(): Correlation Matrix Calculation

The efficient_corr_matrix function computes pairwise correlations between columns of a dataset and visualizes them using a heatmap.

# Example: Compute correlation matrix for mtcars dataset
result = efficient_corr_matrix(mtcars[, 1:5], method = "pearson")

# Print the correlation matrix
result$matrix

Correctness test

Comparison Against R’s cor() Function In this section, we compare the cor_new() function with the built-in cor() and function under C++ framework to verify both correctness and efficiency.

# Correctness Comparison
# Example data
x = c(1, 2, 3, NA, 5)
y = c(2, 4, NA, 8, 10)

# Pearson correlation with mean imputation
custom_result = cor_new(x, y, method = "pearson", na_handle = "complete")
original_result = cor(x, y, method = "pearson", use = "complete.obs")
rcpp_result = cor_new_rcpp_wrapper(x, y, method = "pearson", na_handle = "complete")

# Check if the results are equal
all.equal(custom_result, original_result, rcpp_result)
#> [1] TRUE

Performance Comparison

Interpretation of performance comparison

Three functions were tested for performance: cor_new, cor, and cor_new_rcpp_wrapper. Based on the results, the in-built cor function was the fastest, while cor_new_rcpp_wrapper outperformed cor_new significantly but was still slower than the in-built cor function. The reason for this difference is that cor_new_rcpp_wrapper, implemented in C++, still relies on loops to calculate correlation coefficients. This design choice(choosing to use loops in C++) was made intentionally to facilitate a speed comparison between functions implemented in R and C++.

Summary

The corr.package is a powerful tool for exploring and analyzing relationships in data. With its flexible functions, you can:

Perform advanced correlation analyses with cor_new. Preprocess datasets using handle_na. Efficiently compute and visualize high-dimensional correlation matrices with efficient_corr_matrix. For further information, please refer to the package documentation or help files.
